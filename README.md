# SIGTRAD-UNAP – Proyecto Integrador (Ingeniería de Software)

**Sistema de Gestión de Trámites Documentarios para la UNAP (SIGTRAD-UNAP).**  
Este repositorio contiene el análisis, modelamiento y diseño del sistema (no implementación).

## Objetivo
Digitalizar el ciclo de trámite documentario: **registro → derivación → atención/observación → subsanación → cierre**, con trazabilidad, notificaciones y reportes (SLA).

## Cómo usar los diagramas
- **BPMN**: abrir `docs/01-modelamiento-negocio/01-proceso-ASIS.bpmn` en Draw.io o Bizagi y exportar a PNG.
- **UML (PlantUML)**: abrir archivos `.puml` y exportar a PNG (VSCode + PlantUML o plantuml.com).

## Estructura
- `docs/01-modelamiento-negocio/`: actores, reglas, BPMN AS-IS y TO-BE
- `docs/02-requerimientos/`: RF/RNF, criterios de aceptación, trazabilidad
- `docs/03-uml/`: casos de uso, actividades (≥3), clases, secuencias (≥3), componentes
- `docs/04-modelos-metodologias/`: comparativo de enfoques y recomendación
- `docs/05-patrones-diseno/`: patrones GoF aplicados (≥3)
- `docs/06-scrum/`: backlog (20 HU), historias completas, sprints (≥2), DoD
- `docs/07-investigacion/`: estado del arte + referencias
- `docs/08-proyeccion-social/`: diagnóstico + propuesta + riesgos
- `srs/`: SRS en Word
- `informe-final/`: informe final en Word
